Godot 4: Build Your First 2D Platformer

From installing the engine to exporting a small, playable platformer — using current Godot 4.6 best practices and verified video lessons.

For: total beginner Goal: ship a small 2D platformer Version: Godot 4.6.x (stable, as of June 2026)

This tutorial takes you from zero to a tiny but complete 2D platformer: a character that runs and jumps, a tiled level, coins to collect, an enemy to avoid, a score counter, and an exported build you can hand to a friend. Work through it in order — each step builds on the last, and there's a "ready to move on when…" check so you know you've actually got it.

The spine of this course is one outstanding, free, hour-and-a-quarter video by Brackeys that builds exactly this kind of game. We embed it below and deep-link to the relevant chapter in each step. The written steps add the 2026 context the video can't — what's changed in Godot since it was recorded, and the pitfalls beginners actually hit.

What you need: a Windows, macOS, or Linux machine. Godot is a free, ~120 MB download with no installer and no account. That's it — no paid licenses, ever.

1. Install Godot 4 and create a project

Getting the engine running and understanding the project structure.

Download the Godot 4 Standard build (not the .NET/C# build — you want plain GDScript as a beginner) from the official site, unzip it, and run the executable directly. There's no install step; the engine is the unzipped file.

  1. Go to godotengine.org/download and grab Godot Engine 4.6.x — Standard.
  2. Unzip anywhere and double-click the executable.
  3. In the Project Manager, click + Create, name it, pick an empty folder, and choose the Forward+ renderer (fine for 2D; Compatibility if you're on old hardware or targeting web/mobile).
  4. Click Create & Edit.

2. The core idea: scenes, nodes, and the tree

The single mental model that makes all of Godot click.

Everything in Godot is a node — a small object that does one thing (draws a sprite, plays a sound, detects collisions). You arrange nodes into a tree, and a saved tree is a scene. A scene can be dropped inside another scene as a reusable building block. Your Player is a scene; your Coin is a scene; your Level is a scene that contains many of them.

3. Build the player scene

Creating a controllable character body with a sprite and a collision shape.

Make a new scene with a CharacterBody2D root. Add two children: a Sprite2D (assign it a texture) and a CollisionShape2D (give it a capsule or rectangle shape sized to the sprite). Save it as player.tscn.

In Godot 3 this node was called KinematicBody2D. In Godot 4 it's CharacterBody2D, and the movement code changed too (see the next step). Any pre-2023 tutorial using KinematicBody2D is on the old engine — the concepts carry over, but the node names don't.

I'm confident on the rename; treat the exact movement-API specifics in Step 4 as worth a quick spot-check against the docs (flagged at the bottom).

4. Movement and jumping

Input actions, a movement script, gravity, and jumping.

First define inputs so you're not hard-coding keys. Open Project → Project Settings → Input Map and add actions named move_left, move_right, and jump; bind keys (and a gamepad button if you like) to each. Code then refers to the action, so remapping later is free.

Attach a script to the player. The Godot 4 shape looks like this (gravity pulls you down each frame; pressing jump sets an upward velocity only when you're on the floor):

extends CharacterBody2D

const SPEED = 200.0
const JUMP_VELOCITY = -400.0

func _physics_process(delta):
    # gravity
    if not is_on_floor():
        velocity += get_gravity() * delta
    # jump
    if Input.is_action_just_pressed("jump") and is_on_floor():
        velocity.y = JUMP_VELOCITY
    # left / right
    var dir = Input.get_axis("move_left", "move_right")
    velocity.x = dir * SPEED
    move_and_slide()

Two Godot-4 specifics that trip people coming from older tutorials: move_and_slide() takes no arguments now (it reads the node's velocity property), and gravity is read with get_gravity(). In Godot 3 you passed velocity into move_and_slide(velocity).

If your character falls through the floor or won't move, the usual causes are: no CollisionShape2D on the player or the floor, the script attached to the wrong node, or an input action name that doesn't exactly match the string in your code ("jump" ≠ "Jump").

5. Build the world with tiles

Painting a level with a tileset and giving it collision.

A platformer level is built from tiles: small reusable images you paint onto a grid. You create a TileSet resource from a sprite sheet, mark which tiles are solid (collision), then paint your level with the tile editor in the bottom panel.

Important and easy to miss: as of Godot 4.3 (Aug 2024) the old TileMap node is deprecated. New projects should use one TileMapLayer node per visual layer, all sharing one TileSet. The Brackeys video predates this and uses the old TileMap node — it still works, but the editor will show a "deprecated" warning. Either right-click the TileMap and choose the converter, or just start with a TileMapLayer node from the beginning. Official class reference: TileMap (deprecated); migration notes: Upgrading to 4.3.

The single most common beginner tilemap mistake: forgetting to add physics/collision to your solid tiles in the TileSet editor. Without it, your beautifully painted ground is just a picture and the player drops straight through.

6. Coins and enemies with Area2D signals

Detecting overlaps and reacting to them with signals.

Coins and enemies use Area2D — a node that detects when something enters it, without pushing it around like a solid body would. When the player's body enters a coin's area, the coin emits a signal; you connect that signal to a function that adds to the score and deletes the coin. Same idea for an enemy, except the function ends the run instead.

To delete the collected coin, call queue_free() on it — that safely removes the node at the end of the frame.

7. Dying, score, UI text, and sound

Game state, on-screen UI, and audio feedback.

Put UI on a CanvasLayer so it stays fixed on screen while the camera moves. A Label shows the score; update its text whenever the score changes. For "dying," reload the current scene to restart (get_tree().reload_current_scene()) once the player touches an enemy or falls off the map. Add an AudioStreamPlayer and call play() for jump and coin sounds — small touches that make it feel like a game.

8. Export a playable build

Turning your project into something you can share.

Go to Project → Export. The first time, Godot will prompt you to download the matching export templates (one click). Add a preset for your platform — Windows Desktop, macOS, Linux, or Web — then Export Project to produce a runnable file (or, for Web, an HTML/JS bundle you can host).

Web export gotcha: a Web build won't run by double-clicking the HTML from your file system — browsers block it. It needs to be served over HTTP(S). For a quick local test, run a tiny static server in the export folder; to share it, upload to a host like itch.io.

Common pitfalls & gotchas

  • Wrong download. Grab the Standard build, not .NET, unless you specifically want C#. The .NET build needs the .NET SDK installed and confuses beginners.
  • Following an old-version video blindly. If a tutorial uses KinematicBody2D or passes an argument to move_and_slide(), it's Godot 3. If it uses a single TileMap node without warnings, it predates 4.3. The ideas transfer; the exact node names and a few function signatures don't.
  • Forgetting collision shapes. No CollisionShape2D = the object is a ghost. This is the #1 "why won't it work" for new users.
  • Action-name typos. Input action strings are case-sensitive and must match exactly between the Input Map and your code.
  • Not saving scenes. Unsaved scenes can't be instanced into other scenes. Save early, save often (Ctrl/Cmd+S).

Glossary

Node
The basic building block — one object that does one job (sprite, sound, collider, etc.).
Scene
A saved tree of nodes. Can be reused and nested inside other scenes (.tscn file).
Scene tree
The live hierarchy of all nodes currently running in your game.
GDScript
Godot's built-in, Python-like scripting language. The default and easiest way to start.
CharacterBody2D
The Godot 4 node for player/enemy bodies you move with code (was KinematicBody2D in Godot 3).
Area2D
A node that detects overlaps and emits signals, without physical pushing. Used for pickups, triggers, hurtboxes.
Signal
An event a node emits (e.g. body_entered) that other nodes can connect functions to.
TileMapLayer
The Godot 4.3+ node for painting a grid of tiles. One per visual layer, sharing a TileSet.
TileSet
A resource defining your reusable tiles and which ones have collision.
queue_free()
Safely deletes a node at the end of the current frame.
